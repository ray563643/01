# Codex 維護指南

## 專案性質

這是靜態前端專案，目前沒有 build step。主入口是 `index.html`，邏輯集中在 `app.js`。

## 修改原則

- 優先維持純前端可直接開啟的特性，除非明確要導入框架。
- 人員、排班、店休、需求人數的核心資料結構在 `app.js`。
- 視覺樣式集中在 `styles.css`。
- 固定月份排班圖由 `generate-zero-waste-calendar.ps1` 產生 PNG。
- 修改排班規則後，需重新檢查：正職排假天數、指定休命中、大仁與 Ivy 不同休、每日人數落在需求至需求 + 1、店休不計需求。

## 建議下一步

1. 將目前純前端版改成 Vite + React 專案。
2. 加入 Supabase 或 Firebase，讓手機與電腦同步。
3. 將 2026 年 5 月排班圖產生器整合進主程式，讓使用者按按鈕匯出 PNG。
